== Environment variables

The script makes use of the following env vars:

=== custom vars

* `NOMD`: Set to truthy values (anything except `false` or `0`) to disable the markdown formatting in the console output.
* `TNOSTREAM`: Set to truthy values (anything except `false` or `0`) to force the script to wait for the full response instead of streaming output to the terminal in real-time.
* `REASONING_CONTENT`: Set to any value to show the AI's reasoning blocks (`<think>`) in the console.
* `TELOCITYKEY`: Your API key for the models. Serves as a fallback if you don't explicitly pass `--apikey` on the command line or have it hardcoded in your config.
* `TELOCITY_ERRLOG`: Set to any value to enable verbose logging of HTTP API responses on failure (such as status codes and raw error payloads). Useful for diagnosing upstream rate limits or provider errors.
* `TDEVELOPER`: Set to truthy values (anything except `false` or `0`) to unlock developer-specific presets (such as `internaltranslation`) and tools during configuration load.
* `DEBUG`: Set to any value to enable full, detailed TypeScript stack traces in the terminal output when any command fails.
* `VERBOSE`: Set to any value to also enable detailed stack traces in the terminal output on failure, matching standard CLI verbosity conventions.

=== commonly defined vars

* `NO_COLOR`: Disables ANSI colors in the terminal. (Note: using this also automatically disables markdown formatting).
* `EDITOR`: Used by the `cfg -e` command to figure out which terminal text editor to open your config file in.
* `LANG`: Evaluated on startup to auto-detect and configure the localized system translation strings if no manual locale preference exists in `locale.json`.
