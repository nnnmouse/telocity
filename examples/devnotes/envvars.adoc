= Developer Environment Variable Reference

== 1. EDITOR
* *Evaluation Rule:* String value (executable name or path).
* *Purpose:* Determines the user's preferred command-line text editor. Used as the primary choice for the `cfg -e` command before falling back to a hardcoded list of editors.
* *First Used In:* `configcommand.ts` -> `CfgCommand.execute(argv: string[])`

== 2. LANG
* *Evaluation Rule:* String value (locale string, e.g., `en_US.UTF-8`).
* *Purpose:* Used to detect the user's system locale to automatically select the correct display language for UI strings.
* *First Used In:* `context.ts` -> `AppStateSingleton.getLocale()`

== 3. NO_COLOR
* *Evaluation Rule:* Presence check (`!!process.env["NO_COLOR"]`). Any assigned value (even `"0"` or `"false"`) makes it truthy.
* *Purpose:* Standard environment variable to disable colored ANSI output in the terminal. If truthy, it forces all color functions to act as no-ops and simultaneously disables Markdown rendering.
* *First Used In:* `CLI.ts` (module scope to export styling functions).
* *Reused In:* `context.ts` -> `AppStateSingleton.init()` to conditionally toggle `_NO_MARKDOWN`.

== 4. NODE_ENV
* *Evaluation Rule:* String equality check (`process.env["NODE_ENV"] !== "test"`).
* *Purpose:* Detects if the program is running in a testing environment. If set to `"test"`, it prevents `exitOne()` from overriding the system exit code (`process.exitCode = 1`).
* *First Used In:* `context.ts` -> `exitOne()`

== 5. NOMD
* *Evaluation Rule:* Explicit truthy check (`!!nomd && nomd !== "false" && nomd !== "0"`).
* *Purpose:* Custom flag to entirely disable markdown formatting in terminal outputs. Evaluates to true for any value *except* `"false"` or `"0"`.
* *First Used In:* `context.ts` -> `AppStateSingleton.init()`

== 6. REASONING_CONTENT
* *Evaluation Rule:* Presence check (`!!process.env["REASONING_CONTENT"]`).
* *Purpose:* Determines whether to render or hide the reasoning blocks (`<think>`) generated by AI models. If present, reasoning chunks are piped to the terminal.
* *First Used In:* `oneshotcommand.ts` -> `OneShotCommand.execute()`
* *Reused In:* 
  * `LLM.ts` -> `OutputRenderer` constructor & `OutputRenderer.processItem()`
  * `LLMIO.ts` -> `LLMJSONLBatcher.execute()` (via `streamCallback`)

== 7. TELOCITYKEY
* *Evaluation Rule:* String value.
* *Purpose:* Fallback API key for Large Language Model network requests when `--apikey` is not passed via CLI or defined in the configuration file.
* *First Used In:* `oneshotcommand.ts` -> `OneShotCommand.execute()`
* *Reused In:* `runjsonlcommand.ts` -> `RunJSONLCommand.execute()`

== 8. TNOSTREAM
* *Evaluation Rule:* Explicit truthy check (`!!nost && nost !== "false" && nost !== "0"`).
* *Purpose:* Custom flag to forcefully disable real-time streaming of LLM output to the terminal, waiting for the full response instead. Evaluates to true for any value *except* `"false"` or `"0"`.
* *First Used In:* `context.ts` -> `AppStateSingleton.init()`

== 9. TELOCITY_ERRLOG
* *Evaluation Rule:* Presence check (`!!process.env["TELOCITY_ERRLOG"]`).
* *Purpose:* Instructs the network client to write raw response bodies and status codes to standard error when HTTP API calls fail (return non-2xx status codes).
* *First Used In:* `LLM.ts` -> `LLM.makeRequest()`

== 10. TDEVELOPER
* *Evaluation Rule:* Explicit truthy check (`!!tdev && tdev !== "false" && tdev !== "0"`).
* *Purpose:* Determines if the program is running in developer mode. If false or unset, certain internal developer-only presets (specifically `"internaltranslation"`) are programmatically omitted from the saved default user configuration (`config.json`) upon initialization.
* *First Used In:* `context.ts` -> `AppStateSingleton.init()`
* *Reused In:* `config.ts` -> `configInit()`

== 11. DEBUG
* *Evaluation Rule:* Presence check (`!!process.env["DEBUG"]`).
* *Purpose:* When set, instructs the global error catcher to print raw error stacks to standard error on failure.
* *First Used In:* `main.ts` -> `main()`

== 12. VERBOSE
* *Evaluation Rule:* Presence check (`!!process.env["VERBOSE"]`).
* *Purpose:* Serves as a standard alternative to `DEBUG` for dumping command execution stack traces to standard error on failure.
* *First Used In:* `main.ts` -> `main()`
